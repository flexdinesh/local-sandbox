# Issue 008: Add Codex Run Path

Type: AFK

## Parent

`.scratch/add-codex-harness/PRD.md`

## What to Build

Add the explicit Codex run path. `cbox run codex` should run the Codex Sandbox Image in foreground interactive mode with the caller's Mounted Workspace at `/workdir` and the host Codex home mounted read-write into the container.

The Docker argv generated by the Go CLI should remain equivalent to the Codex Manual Docker Commands, including pass-through commands after the image name.

## Acceptance Criteria

- [x] Manual Docker Commands document running the Codex Sandbox Image with the Mounted Workspace at `/workdir`.
- [x] Manual Docker Commands document mounting the entire host Codex home directory read-write into the container.
- [x] Manual Docker Commands document a Codex pass-through example using `codex --version`.
- [x] `cbox run codex` runs the Codex Harness using Docker argv equivalent to the documented command.
- [x] Codex run argv includes `-it`, `--rm`, the Mounted Workspace mount, and working directory `/workdir`.
- [x] Codex run argv mounts the host Codex home directory read-write into the container.
- [x] Codex run argv does not include named Codex volumes.
- [x] Codex run argv does not pass authentication, token, API key, or proxy environment variables by default.
- [x] `cbox run codex -- codex --version` appends `codex --version` unchanged after the image name.
- [x] Pass-through container commands still require `--`.
- [x] No host bind-mount source prevalidation is performed for Codex.
- [x] Docker/container exit codes are preserved for Codex runs.
- [x] Tests cover Codex run argv, explicit run behavior, pass-through behavior, mount behavior, validation, and exit-code preservation.
